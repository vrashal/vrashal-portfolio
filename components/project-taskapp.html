01 — Overview

What We Built

A lightweight yet powerful Kanban task management application. Tasks can be dragged between "To Do", "In Progress", and "Done" columns with HTML5 Drag and Drop. All data is persisted to Firebase Firestore, enabling real-time updates across browser tabs and devices.

The UI is built in Vue.js with Tailwind CSS for rapid, utility-first styling. Authentication is handled via Firebase Auth, supporting Google OAuth and email/password sign-in.

02 — Key Features

What It Does

🖱️

Drag & Drop

Native HTML5 Drag and Drop API — no external library. Smooth animations on drag start, over, and drop events.

🔥

Real-Time Sync

Firebase Firestore onSnapshot listeners push updates instantly across all open tabs and devices.

🔒

Auth & Boards

Per-user boards secured with Firestore security rules. Google OAuth and email/password sign-in.

🏷️

Labels & Priorities

Colour-coded labels and priority flags (Urgent, Normal, Low) to keep tasks organised at a glance.

📅

Due Dates

Optional due dates with overdue highlighting. Tasks sort automatically by urgency within columns.

📤

Export to CSV

One-click CSV export of all tasks including metadata, for use in external spreadsheets or reporting tools.

04 — Process

Challenge & Solution

The Challenge

Firestore's real-time listeners and Vue's reactive state caused race conditions when dragging tasks simultaneously — the UI would flicker and occasionally revert to the previous order.

The Solution

Implemented an optimistic UI pattern: update local state immediately on drop, then write to Firestore. If the write fails, the local state rolls back. Firestore listeners are temporarily suspended during active drag operations using a debounced flag.
